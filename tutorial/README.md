# Huly туторіал для команди (українською)

Внутрішній навчальний сайт для нашої агенції — як перейти з **Linear** на **Huly**.
Однофайловий, самодостатній (жодних зовнішніх залежностей, шрифтів чи CDN), світла й темна теми.

- **Файл:** [`index.html`](index.html)
- **Скриншоти:** у теці [`img/`](img/) (потрібні для `index.html`)
- **Наш інстанс Huly:** https://huly.kdnx.cloud/workbench/kdnagency/time

## Про скриншоти

У гайді використані скриншоти **справжнього інтерфейсу Huly** з офіційної
документації [docs.huly.io](https://docs.huly.io) (світла тема). Кожен підписаний
із зазначенням джерела.

Хочете показати саме наш інстанс? Замініть файли в `img/` на свої скриншоти з
`huly.kdnx.cloud` (ті самі імена — `create-project.webp`, `create-issue.webp`,
`issue-detail.webp`, `status-board.webp`, `milestones.webp`), і нічого в HTML
міняти не доведеться.

## Що всередині

Покроковий туторіал українською:

1. Навіщо Huly замість Linear
2. Словник відповідностей Linear → Huly
3. Перший вхід
4. Як усе влаштовано (Workspace → Проєкт → Задача)
5. Крок за кроком: проєкти, задачі, статуси, компоненти/milestones, планер
6. Мітки, пріоритети, фільтри; інші модулі; часті питання; чекліст переходу

## Як переглянути

Просто відкрийте `index.html` у браузері — все працює локально.

## Як розмістити на нашому хості

Це статичний файл, тож підійде будь-який спосіб. Наприклад, віддавати його через nginx:

```nginx
location /tutorial/ {
    alias /path/to/huly-selfhost/tutorial/;
    index index.html;
}
```

Після цього гайд буде доступний, наприклад, за адресою `https://huly.kdnx.cloud/tutorial/`
(конкретний шлях залежить від вашої конфігурації nginx / reverse proxy).

## Оновлення

Весь вміст — у `index.html` (HTML + вбудований CSS + невеликий скрипт для навігації та теми).
Скриншоти лежать поруч у `img/`. Редагуйте прямо у файлі; збірка не потрібна.
